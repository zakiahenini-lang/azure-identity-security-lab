# Access Governance

## Purpose

This document explains the access governance model used in the Azure Identity Security and Access Governance Lab.

Access governance helps organizations control, review, and remove unnecessary access to cloud resources and administrative roles.

---

## What Is Access Governance?

Access governance is the process of managing who has access to what, why they have that access, and whether they still need it.

In cloud environments, access can change quickly. Users may change roles, join new teams, or leave an organization.

Without regular reviews, users may keep access they no longer need.

---

## Governance Goals

The goals of access governance in this lab are:

- Review privileged access regularly
- Reduce unnecessary permissions
- Protect administrator roles
- Document access decisions
- Support least privilege
- Improve audit readiness

---

## Access Review Scope

The lab focuses on reviewing access for:

- Cloud Administrators group
- Security Auditors group
- Developers group
- Privileged role assignments
- Conditional Access policy assignments

---

## Example Access Review Process

```text
1. Identify sensitive groups and roles.
2. Review group membership.
3. Confirm whether each user still needs access.
4. Remove unnecessary access.
5. Document the review result.
6. Repeat the review regularly.
